De Hupber API Analyse

Onderzoeksvraag

Hoe kan een Learn To Earn model toegevoegd worden aan de bestaande Handpicked Leeromgeving?


Context

Na het concept idee te hebben gepresenteerd bij de stakeholders, werd er geadviseerd om het concept te presenteren bij de beheerster van de bestaande Handpicked Agencies Leeromgeving. Hierdoor kon er bepaald worden of het concept idee mogelijk was om toe te passen en kon er eventuele feedback worden gevraagd.


Er werd een interview gehouden met D. van der Glas, de beheerster van de Handpicked leeromgeving. Tijdens dit interview werd er geconstateerd dat het niet mogelijk was om het concept idee toe te passen op de bestaande leeromgeving. Wel was het mogelijk om met de API van de leeromgeving een aparte applicatie te kunnen maken. Met deze informatie werd er vervolgens gebrainstormd over allerlei verschillende concept ideeën waar de API gebruikt kon worden. Het doel van de opdracht was namelijk om een Learn To Earn model toe te passen op de bestaande leeromgeving, waarvoor de API voor nodig was.


Methodes

Voor dit onderzoek zijn de volgende CMD-methodes gebruikt.


Expert Interview

Om erachter te komen hoe een Learn To Earn model toegevoegd kan worden aan de bestaande leeromgeving, is er een interview gehouden met de beheerster van de leeromgeving. Zij gaf feedback en kennis over de leeromgeving en vertelde wat wel en niet mogelijk was.


Literature Study

De functionaliteiten van de Hubper API werden geanalyseerd en getest.


Resultaat

De Hubper Api heeft de volgende functies die relevant kunnen zijn naar het idee van tokens en diploma’s verdienen. Deze functies hebben namelijk te maken met de examens en gebruikers:


  • ExamAttempts
    • Haalt het aantal examen pogingen op en kan dit updaten
      • Hiermee kun je een waarde geven aan hoeveel tokens je een gebruiker geeft.
  • Functions
    • Je kunt hiermee functions opvragen en maken
      • Dit zou handig kunnen zijn omdat je potentiële functies zou kunnen maken om te sturen naar Hubper
  • Qualifications
    • Hiermee kun je qualifications zoeken, ophalen en updaten.
      • Dit zou handig zijn voor het zoeken van het zoeken van bestaande qualifications, welke beschikbaar zijn.
  • Qualifications for users
    • Hiermee kun je qualifications ophalen van een gebruikers
      • Hiermee kun je ophalen of een gebruiker een qualification heeft voltooid, dat je vervolgens kunt overzetten naar tokens of diploma’s.
  • User Qualifications
    • Lijst ophalen van de qualifications van alle users
  • Users
    • Hiermee kun je gebruikers ophalen, verwijderen, updaten, activiteiten zien.
      • Gebruiken om een gebruikers profiel op te halen en hiermee kun je ook zien wat de gebruiken allemaal voltooid.

Je kunt geen cursussen ophalen of daar punten aan geven. Wel kun je gebruiker gegevens ophalen.


Conclusie

Met deze functie zou er potentieel een applicatie gemaakt kunnen worden dat de gebruikers gegevens omzet naar diploma’s en tokens. Hierdoor blijft de leeromgeving hetzelfde en komt er een ‘upgrade’ bij die een Learn To Earn element toevoegt.


Schets van hoe de applicatie potentieel zou kunnen werken.


Echter is het niet mogelijk om de API persoonlijk voor dit project te gebruiken, vanwege privacy redenen. Dit kan worden opgelost door de API te vervangen met een json dummy data. Echter betekent dit dat de gebruikerstesten low fidelity moeten gaan zijn, omdat er zonder de API-toegang er geen manier is om een high fidelity prototype te testen op gebruikers ervaring. De gebruikers ervaring kan wel getest worden om met de Wizard Of Oz techniek de applicatie te simuleren en zo te testen.


Vervolg

Met de conclusie kon er een tweede concept idee worden gemaakt. Dit concept idee kon vervolgens weer gepresenteerd worden aan de stakeholders en beheerster om zo tot een officieel concept idee te leiden die volledig wordt uitgewerkt en getest.
